# Customer Acquisition Paths

The early acquisition motion should be narrow and manual:

> Find teams with existing AI usage, offer a free Kimi + GPT-5.5 routing assessment, obtain sanitized samples, produce a one-page savings report, then sell a 7-day pilot deployment.

## Path 1: AI Tool Site Outreach

This is the highest-priority path.

### Where To Find Leads

Use AI tool directories and search engines:

- AIGC tool directories.
- AI writing and office-tool directories.
- Product websites listed in AI navigation sites.
- Search engines with Chinese AI tool keywords.

Search terms:

```text
AI 写作工具
AI 文案生成
AI 客服系统
AI 知识库问答
AI 简历优化
AI 论文润色
AI 商品文案
AI 小红书文案
AI 公众号写作
```

### Screening

Prioritize products that:

- Have an AI text, Q&A, customer support, or content-generation feature.
- Have login, subscription, paid credits, or pricing.
- Have public contact information.
- Appear to be a small or mid-sized team.
- Likely have real model costs.

Skip:

- Pure free toys.
- Large platforms.
- Image/video-only tools.
- Sites with no contact path.
- Sites that market grey-market API access.

### Daily Workflow

```text
09:30-10:30 collect 20 new leads
10:30-11:30 send 15-20 customized messages
14:00-15:00 follow up old leads
16:00-17:30 run discovery, prepare reports, or demo routing
```

Weekly target:

- 100 new leads.
- 80 outreaches.
- 8-15 effective replies.
- 3-5 diagnostic conversations.
- 1-3 sample sets.

## Path 2: Dify, FastGPT, RAG, And AI Implementation Ecosystem

These are channel leads. They already serve end customers and need repeatable model infrastructure.

### Where To Search

```text

RAG 企业知识库
企业 AI 助手开发
AI 智能客服开发

RAGFlow 部署
```

Look on:

- Bilibili.
- CSDN.
- Zhihu.
- GitHub.
- WeChat public articles.
- Developer communities.

### What To Offer

Offer a reusable gateway layer:

- Unified OpenAI-compatible endpoint.
- Customer/project keys.
- Kimi/GPT-5.5 routing.
- User-level usage.
- Budget limits.
- Monthly billing reports.

Suggested commercial models:

| Model | Pricing |
|---|---:|
| Single project gateway base | RMB 5,000-30,000 |
| White-label base fee | Fixed base + partner markup |
| Monthly managed revenue share | 20%-40% to partner |

## Path 3: GitHub Developer Prospecting

Use GitHub to find technical buyers with active projects.

Search terms:

```text

openai api writing tool
kimi api chatbot
llm gateway
knowledge base openai api
customer support chatbot openai
```

Look for:

- Recently updated repositories.
- Demo or commercial website.
- Issues mentioning API cost, key management, usage, or model selection.
- Public contact information.

This path has lower volume but higher technical relevance.

## Path 5: Cross-Border Ecommerce And Content Teams

This path is useful because a Singapore invoice may be acceptable to some cross-border or overseas-facing teams.

Find:

- Amazon sellers.
- Shopify agencies.
- Cross-border ecommerce operators.
- Overseas marketing agencies.
- Listing optimization teams.
- Content operations teams.

Do not lead with API gateway. Lead with workflow:

> Kimi handles bulk product information processing and drafts; GPT-5.5 handles final sales copy and quality review; the gateway tracks team usage and cost.

Entry offers:

| Offer | Price |
|---|---:|
| 100 product-copy workflow trial | RMB 999-3,000 |
| Content workflow deployment | RMB 5,000-20,000 |
| Monthly managed workflow | RMB 2,000-8,000/month |

## Path 6: Technical Content

Content is support infrastructure, not the only acquisition channel.

Publish 2 technical posts per week:

- Kimi + GPT-5.5 routing.
- API cost control for AI SaaS.
- User-level token billing.
- OpenAI-compatible gateway design.
- Budget alerts and abuse controls.

Each post should end with:

> If your AI product already spends more than RMB 3,000/month on model APIs, send 10 sanitized request samples and we will assess which requests can move to Kimi and which should stay on GPT-5.5.
