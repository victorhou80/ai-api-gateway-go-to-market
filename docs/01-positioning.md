# Positioning

## Core Position

The product should not be positioned as an API recharge service or an API reseller.

Use this positioning:

> A Chinese-business-focused AI API gateway that routes routine work to Kimi for cost control and routes high-value work to GPT-5.5 for output quality, while providing unified access, key management, usage reporting, budget controls, and monthly operations.

The customer buys operational control and business reliability, not raw model access.

## Why Kimi + GPT-5.5 Is Enough For The First Niche

Supporting only two models is a constraint, but it can be turned into a simple story:

- Kimi handles high-volume Chinese text processing, summaries, classification, drafts, customer support knowledge retrieval, and routine content operations.
- GPT-5.5 handles complex reasoning, final drafts, high-value client communication, consulting deliverables, and quality review.
- The gateway decides which model should handle each task.

Do not claim to be a broad multi-model platform until additional providers are truly available.

## Primary Promise

The promise is:

> Reduce unnecessary GPT-5.5 usage without lowering quality on critical outputs.

The first proof point should be a routing assessment:

- Identify what can move to Kimi.
- Identify what must stay on GPT-5.5.
- Estimate cost savings.
- Define routing rules.
- Deploy a controlled pilot.

## What To Sell

Sell these outcomes:

- Lower model cost for routine Chinese-language work.
- More predictable API spend.
- User/project-level usage reporting.
- Safer key management.
- Budget limits and abuse controls.
- Failure retry and fallback.
- Monthly cost and usage reports.

## What Not To Sell

Avoid these phrases:

- API recharge.
- Cheap GPT quota.
- GPT balance top-up.
- API key resale.
- One-dollar credits.
- Region bypass.
- Account rental.

Use these phrases instead:

- AI API gateway.
- Model routing.
- Cost governance.
- Usage analytics.
- Budget controls.
- Customer-owned key support.
- Managed gateway operations.

## First Demo Assets

Prepare these demos before heavy outreach:

1. Content Workflow Demo
   - Input: product or article material.
   - Kimi: draft and structure.
   - GPT-5.5: final sales copy or executive-grade rewrite.
   - Output: cost and quality comparison.

2. Knowledge Assistant Demo
   - Input: product docs or internal policy.
   - Kimi: routine questions.
   - GPT-5.5: complex or high-risk questions.
   - Output: routing log and usage table.

3. API Tool Cost Demo
   - Input: simulated 1,000 requests.
   - Kimi: summary, classification, draft tasks.
   - GPT-5.5: complex and final tasks.
   - Output: all-GPT cost vs routed cost.
